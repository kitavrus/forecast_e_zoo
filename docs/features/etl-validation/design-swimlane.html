Swimlane — ETL & Validation (Модуль 2)

Карта процесса суточного ETL-рана: cron 02:30 Kyiv (или admin POST) → fetch snapshot из source-adapter → cross-validation → построение 5 mart-таблиц → atomic flip в схеме marts. Подписи стрелок раскрывают поля DTO, SQL и Prometheus-метрики.

Наведите курсор на подпись стрелки — увидите, какие именно поля передаются
Бизнес-цели (SLA)
ETL Freshness
etl_runs.committed_at < 03:30 Europe/Kyiv в 99% дней (run ≤ 30 мин)
Marts Completeness
5 mart-таблиц обновлены к 04:00 Europe/Kyiv в ≥98% дней
Cross-entity Quality
lines_failed/lines_total < 0.5% (hard fail при > 1%)

Cross-validation
fk_exists, unique_business_key, aggregate_sum_matches, referential_integrity

demand_history / calculation_input / kpi_daily / master_current / supplier_scorecard
Loader
UPSERT marts.* + atomic flip
Quality gate
Проверка lines_failed/lines_total; при > 1% — fail run без flip
Atomic flip
Один tx: UPSERT 5 marts + UPDATE etl_runs status='committed'
Storage
PG: marts.*, etl_runs, reject_log
Lock + run record
pg_try_advisory_lock(hash('etl-run')); INSERT etl_runs(status='running')
Staging tables
INSERT INTO stg_{entity} (TEMP per run); фиксация source_load_id
reject_log

marts.* commit
5 mart-таблиц + etl_runs.committed_at = now() в одной tx
Metrics emit
etl_run_success_total, etl_lines_processed_total{entity}, mart_rows_total{mart}
Прямой поток — данные, snapshot, mart-строки
Обратный поток — статус, метрики, ошибки 401/409/503/failed
Как читать диаграмму. Каждая строка — роль (Trigger / Extractor / Transformer / Loader / Storage), каждая колонка — фаза (Trigger → Validation → Business logic → Persistence → Response). Задача живёт на пересечении «роль × фаза». Стрелки показывают передачу ответственности: сплошные — прямой поток, пунктирные — обратная связь (статусы, ошибки). Наведите курсор на любую подпись стрелки, чтобы увидеть детальный список полей DTO, SQL-параметров и Prometheus-метрик.
