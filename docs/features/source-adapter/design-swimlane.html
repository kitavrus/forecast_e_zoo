source-adapter — swimlane (роль × фаза)

Гибрид двух поверхностей: фоновый cron-load (Scheduler → ERP → snapshot flip) и синхронный HTTP API (Client → Handler → Service → Repository → PG). Фазы общие, роли совмещены так, чтобы оба flow читались на одной канве.

Как читать диаграмму. Каждая строка — роль, каждая колонка — фаза процесса. Адаптер совмещает два сценария: cron-load (Scheduler → Loader → Repository → PG, фоновый) и HTTP API (Client → Handler → Service → Repository → PG, синхронный). Стрелки показывают передачу ответственности: сплошные — прямой поток, пунктирные — обратный (ошибки, 503/409). Наведите курсор на любую подпись стрелки, чтобы увидеть передаваемые поля и SQL-параметры. Полные DTO/SQL — в design-go-layers.md, design-sql.md, design-sequence-diagrams.md.
