Вариант 4. Декомпозиция по потокам (стрелкам)

Каждая подписанная стрелка общей диаграммы превращается в собственный swimlane: на нём показано, как сообщение проходит между ролями и какие поля/форматы у него на каждой стадии.

Когда удобно

  • Чётко отвечает на «что между двумя системами летает»
  • Удобно для проектирования контрактов и API
  • Тултипы общей диаграммы расширяются в полноценную диаграмму

Минусы

  • Нет «своей» роли — фокус на канале, а не на исполнителе
  • Похожие потоки приходится дублировать (canonical из 1С vs SAP)
  • Сценарии «многих ко многим» (channel-orders) тяжело уложить в один файл
