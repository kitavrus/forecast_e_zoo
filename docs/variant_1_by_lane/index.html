Вариант 1. Декомпозиция по ролям

Каждая дорожка общей диаграммы (роль) раскрывается в отдельный swimlane. Внутри — её подмодули и пять последовательных шагов работы.

Когда удобно

  • Команды собраны вокруг ролей (DevOps X-Flow, ML Replenishment)
  • Нужно показать «зону ответственности» одного юнита
  • Полезно для онбординга в конкретный сервис

Минусы

  • Сквозной путь данных не виден на одном экране
  • Дубль контекста: один поток упоминается в двух разных lane-диаграммах
  • Тяжелее увидеть последовательность фаз
