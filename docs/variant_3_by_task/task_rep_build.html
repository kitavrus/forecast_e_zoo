Прогноз → калькуляция → конструктор

Как читать. Это «зум» в одну ячейку общей диаграммы (Replenishment × Фаза 2). Лейны — пять подмодулей: feature builder → forecaster → net requirement calc → rules engine → constructor. Каждый шаг по горизонтали — следующая стадия трансформации. Пунктир от KPI (внешний контур) приходит в forecaster и rules — это контур обучения и калибровки.
